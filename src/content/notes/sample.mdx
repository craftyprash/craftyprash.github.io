This is a comprehensive sample note demonstrating the note layout with table of contents on the right side. Notes are designed for quick reference articles and one-pagers that you want to access quickly.

## Introduction

Notes are single-page articles with a table of contents on the right side. They're perfect for documenting quick tips, reference materials, or standalone concepts that don't require multiple chapters.

The layout automatically generates a table of contents from your h2 and h3 headings, making it easy to navigate longer articles. The TOC is sticky, so it stays visible as you scroll through the content.

## Key Features

The note layout includes several features designed to enhance readability and sharing:

- **Automatic table of contents generation** - No manual work needed, just write your headings
- **Share buttons for social media** - LinkedIn and X (Twitter) integration
- **Responsive design** - Works beautifully on desktop, tablet, and mobile
- **Clean typography** - Optimized for technical content with proper code formatting
- **Comment system** - Giscus integration for discussions

### Typography and Readability

The content width is optimized at 720px maximum, which provides approximately 66-75 characters per line - the ideal range for comfortable reading. This is based on research showing that line lengths in this range reduce eye strain and improve comprehension.

Text uses a softer gray color (#c9d1d9) instead of pure white, which is easier on the eyes during extended reading sessions. Headings use a brighter white (#e6edf3) to create clear visual hierarchy.

### Code Examples

Code blocks are styled with proper syntax highlighting support. Here's an inline code example: `const example = 'value';`

Block code examples maintain readability with appropriate spacing and contrast.

## Content Organization

When writing notes, consider organizing your content with clear sections. Use h2 for main sections and h3 for subsections. This creates a logical hierarchy that's reflected in the table of contents.

### Best Practices

Here are some tips for writing effective notes:

1. Start with a clear introduction that explains what the note covers
2. Use descriptive headings that make sense out of context
3. Keep paragraphs focused on a single idea
4. Use lists to break down complex information
5. Include code examples where relevant

### Visual Elements

You can include various visual elements to enhance your notes. Images, tables, and blockquotes all have appropriate styling.

This layout is perfect for quick reference articles and one-pagers. The combination of clean typography, optimal reading width, and helpful navigation features creates an excellent reading experience for technical content.
