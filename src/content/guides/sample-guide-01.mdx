Welcome to this comprehensive guide! This is the first chapter where we'll cover the fundamentals and get you started on your journey. Guides are designed for multi-chapter content that builds upon itself progressively.

## Overview

Guides are multi-chapter documents with navigation on the left sidebar and a table of contents on the right. This three-column layout provides excellent navigation while maintaining optimal reading width for the main content.

The guide format is perfect for:

- Step-by-step tutorials that span multiple topics
- Comprehensive documentation with logical progression
- Course materials or learning paths
- Technical documentation that requires multiple sections

The guide layout features three distinct areas, each serving a specific purpose:

### Left Sidebar - Chapter Navigation

The left sidebar displays all chapters in the guide. The current chapter is highlighted with an accent color, making it easy to see where you are in the guide. Click any chapter to jump directly to it.

This sidebar is sticky, meaning it stays visible as you scroll through the content. On mobile devices, it moves to the top of the page to save horizontal space.

### Center Column - Main Content

The main content area is optimized for readability with a maximum width of 720px. This ensures comfortable line lengths regardless of your screen size.

Content includes:

1. A clear title and metadata (date, tags)
2. Share buttons for social media
3. The main article content with proper typography
4. Previous/Next navigation at the bottom
5. Comment section for discussions

### Right Sidebar - Table of Contents

The right sidebar contains an automatically generated table of contents based on your h2 and h3 headings. This helps readers quickly jump to specific sections within the current chapter.

## Navigation Features

Guides include several navigation features to help readers move through the content efficiently.

### Chapter Sidebar

Use the sidebar to jump between chapters. The active chapter is highlighted, and hovering over other chapters shows a preview state. This makes it easy to explore the guide structure.

### Previous and Next Buttons

At the bottom of each chapter, you'll find Previous and Next buttons. These allow linear progression through the guide without needing to use the sidebar. The buttons show the title of the adjacent chapters, giving context about what comes next.

### Table of Contents

The TOC on the right provides quick navigation within the current chapter. Click any heading to jump directly to that section. This is especially useful for longer chapters.

## Writing Effective Guides

When creating guides, consider these best practices:

> Structure your guide so each chapter builds upon the previous one. Readers should be able to follow along linearly and understand concepts progressively.

### Chapter Organization

Each chapter should focus on a specific topic or concept. Avoid trying to cover too much in a single chapter. It's better to have more focused chapters than fewer lengthy ones.

### Clear Headings

Use descriptive headings that clearly indicate what each section covers. This helps both with navigation and with understanding the guide's structure at a glance.
